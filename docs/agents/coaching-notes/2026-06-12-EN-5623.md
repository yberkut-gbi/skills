# Coaching note — EN-5623: fe-verify-ui

## What shipped

New `fe-verify-ui` skill (`skills/engineering/fe-verify-ui/SKILL.md`) — the Playwright arm of the `fe-flow` green gate. Three supporting files updated: `fe-flow/SKILL.md` step 3 (wires it in), `README.md` (skills table + model-split note).

## What went well

- ADR 0001 §6 was precise enough that no scope alignment was needed — the acceptance criteria mapped directly to skill sections.
- The "opt-in by presence" pattern (check `.claude/skills/fe-verify-ui/SKILL.md` exists) is the right portability move: no mandatory dep, no hard-stop in repos that haven't installed it.
- The three-check real-API assertion (zero calls → fail, placeholder text → fail, identical-rows heuristic → fail) is concrete enough to implement without leaving judgment calls to the agent at runtime.

## What to watch in review

- The escape-hatch wording ("skipped (Playwright unavailable) → continue") needs to be understood by `fe-flow` — the current step 3 edit does say "if it reports 'skipped (Playwright unavailable)', continue", so it's explicit. Reviewer should confirm that reads clearly.
- The 80% identical-rows heuristic (§6 check 3) is a reasonable starting point but may produce false positives in paginated lists where the first page happens to be uniform. Consider making this configurable via `config.md` in a follow-up.
- Browser-MCP enrichment (§7) is described as additive only — good — but the detection ("calling it and checking for a non-error response") is a bit hand-wavy. A more precise detection (check for a specific tool name in the available tools list) would be worth specifying if this skill gets promoted to a featured reference.

## Efficiency

Single context, no sub-agents needed — this was pure prompt authoring with three targeted file edits. Token cost was low. The fe-flow update is one sentence; README is two lines. Kept the blast radius minimal.

## Rules to consider distilling

- When adding a new skill that plugs into an existing conductor, the canonical integration pattern is: (1) write the skill, (2) add one sentence to the conductor's relevant step naming the skill and the detection condition, (3) update the registry (README). No other files need touching for a first-class integration.
